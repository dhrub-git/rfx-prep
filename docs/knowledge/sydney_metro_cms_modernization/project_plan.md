# Project Plan (High-Level)

## Phases
- Discovery (2 weeks): workshops, content audit, non-functional requirements
- Architecture & Design (2 weeks): target design, security, migration plan
- Build & Configure (6 weeks): CMS setup, APIs, templates, pipelines
- Migration (4 weeks): ETL, redirects, validation
- Testing & Hardening (3 weeks): perf, accessibility, security
- Cutover & Handover (1 week): go-live, training, support

## Milestones
- M1: Discovery complete; approved NFRs
- M2: Architecture signed-off; migration runbook
- M3: CMS configured; initial content types; CI/CD live
- M4: Migration dry run complete; redirect map validated
- M5: Performance and accessibility tests pass
- M6: Go-live; support runbooks and training delivered

## Roles
- PM, Solution Architect, Lead Dev, Content Migration Lead, SRE, QA, Accessibility SME, Security

## Risks
- Legacy export blockers; new policy changes; peak traffic events
